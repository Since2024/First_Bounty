## 2) 24-hour Prioritization Strategy

### Tier 1 (Do first)
1. End-to-end happy path works reliably.
2. README is complete and easy for judges to run.
3. Demo script is rehearsed and under 90 seconds.

### Tier 2 (Do next)
1. Clean UX labels and flow for first-time users.
2. Add clear error states (wallet missing, bad image, API failures).
3. Add one impact metric slide (e.g., "KYC prep reduced from 20 min to 3 min").

### Tier 3 (If time remains)
1. Lightweight analytics/admin dashboard screenshots.
2. Extra form template(s).
3. Small growth angle (B2B SaaS pricing hypothesis).

## 6) Final Hour Before Submission
1. Freeze features.
2. Re-run app walkthrough once.
3. Re-run tests once.
4. Verify README setup steps from scratch.

## 7) Common Mistakes to Avoid
- Over-explaining architecture but not showing working flow.
- Missing setup steps or broken install command.
- Demo longer than 2 minutes.
- No clear target user / business path.
- Submitting without team roles and clear ownership.
